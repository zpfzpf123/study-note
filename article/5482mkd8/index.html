git各分支开发，最后合并主分支

约 451 字大约 2 分钟

git

2025-02-26

步骤详细描述示例命令注释
重新拉取远程分支获取远程分支的最新信息git fetch origin此命令不会更改本地分支状态，仅更新远程分支的信息
确保在 dev-zhou 分支确认当前所在分支并切换到 dev-zhou 分支git status 或 git branch 查看分支
git checkout dev-zhou 切换分支		git status 查看工作区状态及当前分支
git branch 列出所有分支
拉取远程 develop 分支最新代码保持本地 develop 分支与远程一致git fetch origin develop
git checkout develop
git merge origin/develop 或 git pull origin develop		git fetch 获取远程分支更新
git merge 合并分支
git pull 相当于 git fetch 加 git merge
将 develop 分支代码合并到 dev-zhou 分支把最新的 develop 分支代码合并到 dev-zhou 分支git checkout dev-zhou
git merge develop		切换到 dev-zhou 分支后进行合并操作
解决可能的冲突处理合并时出现的冲突git add <conflicted-file>解决冲突后将文件添加到暂存区
提交合并的更改提交合并的结果git commit -m "Merge develop into dev-zhou"提交时添加合并的描述信息
（可选）推送更改到远程仓库将本地 dev-zhou 分支更改推送到远程git push origin dev-zhou推送本地更改到远程对应分支
将 dev-zhou 分支推给远程 develop 分支先确保 dev-zhou 分支最新，再进行合并和推送git checkout dev-zhou
git pull origin dev-zhou
git checkout develop
git pull origin develop
git merge dev-zhou
git push origin develop		一系列操作确保分支状态正确并完成推送
